Big Integer (Lua)
The Story
During my Introduction to Cryptography course, my professor gave the class the freedom to use the programming language of their choice for the homework assignments. I used this opportunity to learn a new language: Lua. While this was an exciting challenge at the start, it quickly became a problem when we started working with numbers too large to fit in the primitive data types. Most languages have a way of overcoming this problem. For example, Java allows users to declare BigInteger types and Python automatically converts numbers to BigIntegers once they are too large. However, Lua has no native support for any such feature. So, I took it upon myself to create an entirely new data type for Lua, ensuring that every operation possible in Java would be possible in Lua. After typing over 1000 lines of math-intensive code, I was finally able to complete my homework assignments for my Cryptography class using Lua.
The RSA algorithim being used to share the private key "m." Completed using the custom BigInteger datatype, and presented using LÖVE. Numbers p and q were found using the Miller-Rabin primality test, built into the BigInteger source code.
A demonstration of all possible operations for a BigInteger. All numbers used are too large for primitve data types, but my custom data type completed all operations in 1.77 seconds. Click the image to open a larger version.

Features
  • Store integers of any size with exact precision.
  • Speed comprabable to primitve data types for most numbers.
  • Perform basic operations, such as subtraction and division.
  • Perform advanced operations, such as performing a primality test.
  • Perform bitwise operations.
  • Work in number bases up to 36, including binary and hexadecimal.
  • A syntax nearly identical to what is found in Java.
Accomplishments
  • Created a solution that others have used in their projects, getting my code from GitHub.
  • Quickly learned the nuances of a new language in order to complete my goal.
  • Applied arithmetic shortcuts to achieve speeds faster than the intuitive solution has to offer.
  • Allowed for cryptographic algorithims requiring large prime numbers.
Hieroglyphic Minesweeper (Java)
The Story
Over the summer of 2020, during the COVID-19 pandemic, a friend from college challenged me to do the 100 days of code challenge. The gist of the challenge was to code for one hour a day, 100 days in a row. Considering the fact that I was quarantined at home, I accepted the challenge. I used this time to complete some abandonded projects. Among those projects, was a varient of the videogame Minesweeper, where the numbers are replaced by randomized symbols. I originally was going to make the final product text-based, but that was not a user friendly solution. So, I went for the GUI (Graphical User Interface) approach instead. I had a fully playable game in only a couple of days, but I decided to go above my original plans by adding many features that I found lacking from the original Minesweeper. This was entertaining to code, and the end result provided me with a form of entertainment.

Features
  • Customize the unrevealed tile, revealed tile, and text colors.
  • Toggle between icons and numbers for varying difficulty.
  • Create field sizes of up to 40x40 tiles, with any amount of mines.
  • An option to reveal a blank tile, making the first/worst move of the game more tolerable.
  • A summary page at the end of each field, noting details such as how much of the field was cleared.
Accomplishments
  • Became very comfortable with GUI coding in Java.
  • Provided dozens of people with a new challenge in their favorite game.
  • Discovered new optimizations for previous code I have written.
